# EPL 语言语法 Skill

本 Skill 记录自定义 EPL 语言的语法、关键字和代码结构规则，供代理在生成或理解 EPL 代码时遵循。

## 总体规则

- **唯一支持的注释形式**：单引号 `'注释内容`，且**只能出现在 `.子程序` 的代码块内部**。
- **缩进要求**：
  - 条件语句与循环语句体内的代码必须按层级缩进（推荐 4 空格），并且关键字与代码之间按示例换行。
  - 其他语句（定义、普通表达式等）统一不缩进，全部写在行首。
- **关键字以点开头**（如 `.版本`、`.子程序`、`.如果真` 等），必须完整匹配。
- 代码文件整体结构**必须按固定顺序**书写（见下文「程序结构顺序」）。

## 程序结构顺序

EPL 源文件顶层结构，**必须按下面顺序出现**（可省略某些段落，但顺序不可打乱）：

1. `.版本 2`
2. `.支持库 库名`  
   - 仅引用 `sys_libs/<库名>.txt` 中定义的支持库；**不等于 DLL**，不能由代理创建，必须由用户手动维护。
3. `.常量 ...`
4. `.DLL命令 ...`
5. `.数据类型 ...`
6. `.全局变量 ...`
7. `.程序集 ...`
8. `.程序集变量 ...`
9. `.子程序 ...`（函数实现，允许多次出现）

### 入口点约定

典型入口结构：

- `.程序集 程序集N`
- `.子程序 _启动子程序, 整数型`
  - 子程序体中编写实际入口逻辑。
  - 以 `返回 (0)` 结束。

## 数据类型

支持的数据类型：

- `字节型`
- `短整数型`
- `整数型`
- `长整数型`
- `小数型`
- `双精度小数型`
- `子程序指针`

## 运算符

算术与比较运算符（多写法等价）：

- 加：`+` 或 `＋`
- 减：`-`
- 乘：`*` 或 `×`
- 除：`/` 或 `÷`
- 大于：`>` 或 `＞`
- 大于等于：`>=` 或 `≥`
- 小于：`<` 或 `＜`
- 小于等于：`<=` 或 `≤`
- 相等：`==`、`=` 或 `＝`

位运算使用函数形式：

- `位与(a, b)`
- `位或(a, b)`
- `位异或(a, b)`
- `左移(a, b)`
- `右移(a, b)`

逻辑运算：

- `a 且 b`
- `a 或 b`
- `取反(a)`

## 条件语句

### 简单条件：`.如果真`

结构：

```text
.如果真 (条件)
    代码...
.如果真结束
```

要求：

- `.如果真` 与 `.如果真结束` 各自独占一行。
- 条件后必须换行开始代码块，块内代码缩进。

### 多分支条件：`.判断开始 / .判断 / .默认`

这类语句语义上等价于多分支 if/else if/else。

#### 形式一：只有默认分支

```text
.判断开始(条件)
    代码...
.默认
    代码...    ← 此行可以是 4 空格 + 代码，也可以完全为空
.判断结束
```

#### 形式二：含多个条件分支与默认分支

```text
.判断开始(条件1)
    代码...
.判断(条件2)
    代码...
.判断(条件3)
    代码...
.默认
    代码...    ← 此行可以是 4 空格 + 代码，也可以完全为空
.判断结束
```

要求与约束：

- `.判断开始(...)`、每个 `.判断(...)`、`.默认`、`.判断结束` 均需独立成行。
- 各分支内部代码必须缩进（推荐 4 空格）。
- `.默认` 下面可以是 4 空格加代码，也可以完全没有代码与空格。

## 循环语句

### 变量循环：`.变量循环首 / .变量循环尾`

```text
.变量循环首(开始, 结束, 增量, 变量)
    代码...
.变量循环尾()
```

- `变量` 为循环计数变量。

### 计次循环：`.计次循环首 / .计次循环尾`

```text
.计次循环首(次数, 变量)
    代码...
.计次循环尾()
```

### 条件循环：`.判断循环首 / .判断循环尾`

```text
.判断循环首(条件)
    代码...
.判断循环尾()
```

### 跳出循环

- 使用：`跳出循环 ()`  
  - 在任意循环体内部调用，用于提前结束当前最近的循环。

## 变量与常量定义

### 变量定义

通用格式（空字段以逗号占位）：

- 局部变量（只能在 `.子程序` 内）：
  - `.局部变量 名称, 类型, , "数组个数", 注释`
- 程序集变量：
  - `.程序集变量 名称, 类型, , "数组个数", 注释`
- 全局变量：
  - `.全局变量 名称, 类型, , "数组个数", 注释`

### 常量定义

格式：

- 数值常量：
  - `.常量 常量名称, "常量数值", 公开, 常量注释`
- 文本常量：
  - `.常量 常量名称, "“常量文本”", 公开, 常量注释`

访问方式：

- 作为常量表达式：`a = #常量名`
- 作为函数参数：`func(#常量名)`

## 子程序（函数）定义

子程序头与参数、局部变量定义示例：

```text
.子程序 子程序名称, 返回类型, 公开, 注释
.参数 参数名称, 类型, 参考|数组|可空, 参数注释
.局部变量 局部变量名称, 整数型, 静态, "数组个数", 变量注释
    子程序内部代码...
返回 (表达式)
```

要点：

- `.子程序` 头行在行首，不缩进。
- 参数使用 `.参数` 定义，可选修饰：`参考` / `数组` / `可空`。
- `.局部变量` 可指定是否 `静态`，以及数组大小。
- 子程序体内允许使用 `'注释`。

## 指针操作

相关操作及约束：

- 获取变量指针地址：
  - `取指针地址(变量)` → 返回 `整数型`
- 获取子程序指针：
  - `&变量` → 返回 `子程序指针`
- 错误示例：
  - `取指针地址(#常量)` 是 **非法** 的。
  - 如需对常量取指针，必须先将常量赋值到变量，再对该变量调用 `取指针地址(变量)`。

## DLL 调用定义

EPL 使用 `.DLL命令` 定义外部 DLL 调用，支持 `stdcall`（默认）与 `cdecl` 两种调用约定。

### stdcall（默认）

```text
.DLL命令 FFI_名称, 返回类型, "dll名", "函数名", 公开, 注释
.参数 参数名称, 类型, 传址|数组, 参数注释
```

### cdecl（函数名前加 @）

```text
.DLL命令 FFI_名称, 返回类型, "dll名", "@函数名", 公开, 注释
.参数 参数名称, 类型, 传址|数组, 参数注释
```

说明：

- `FFI_名称` 为 EPL 内部调用名。
- 参数必须通过 `.参数` 定义，参数传递方式使用 `传址` 或 `数组`。

## 结构体（自定义数据类型）

使用 `.数据类型` 与 `.成员` 定义结构体：

```text
.数据类型 自定义数据类型名称, 公开, 自定义数据类型注释
.成员 成员名称, 类型, 传址, "数组个数", 成员注释
```

说明：

- `.数据类型` 定义类型本身。
- `.成员` 定义结构体字段，指定类型、是否传址以及数组大小。

## 生成 EPL 代码时的注意事项

1. **严格遵守程序结构顺序**：从 `.版本` 起，后续段落顺序固定。
2. **条件与循环必须缩进与换行**：关键行（如 `.如果真`、`.判断开始`、`.变量循环首` 等）单独占行，代码块内统一缩进。
3. **除条件/循环代码块外，其余语句一律顶格**：包括 `.子程序` 头、变量/常量/DLL/数据类型定义等。
4. **注释范围限制**：仅在 `.子程序` 内部使用 `'注释`，不要在顶层结构或定义行使用注释。
5. **常量取指针限制**：对常量取指针必须经过中间变量。
